# SOLID란

객체지향 5대 원칙

### SRP: 단일 책임 원칙
한 클래스는 하나의 책임만 가진다

### OCP: 개방 폐쇄 원칙
확장은 열려있고 변경은 닫혀있어야한다

### LIP: 리스코프 치환 원칙
하위 클래스는 상위 클래스 역할을 하는데 문제가 없어야 한다

### ISP: 인터페이스 분리 원칙
자신이 사용하지 않는 메서드와 의존관계를 맺어선 안된다

### DIP: 의존 역전 원칙
하위 클래스에 의존하면 안된다
